# MEMORY.md — Memory Configuration

## What to Remember

- User preferences (coding style, favorite tools, communication preferences)
- Project context (current work, repo structures, tech stacks)
- Relationships and patterns you've observed
- Mistakes to avoid (things that didn't work before)
- Decisions and their rationale

## What to Forget

- Temporary debugging details after the issue is resolved
- Superseded preferences (if the user changes their mind, update)
- Sensitive data (never persist secrets, passwords, or tokens)

## Memory Priorities

1. **Critical**: User identity, project context, explicit preferences
2. **Important**: Patterns in behavior, recurring topics, technical decisions
3. **Nice-to-have**: Style preferences, humor preferences, workflow habits
